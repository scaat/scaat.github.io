Linux学习注意事项

发表于 2019-03-05 | 分类于 基础技能
本文字数： 984 | 阅读时长 ≈ 1 分钟

  • Linux 严格区分大小写

    与 Windows 不一样，Linux 是区分大小写的，包括文件名、目录名、命令选项、配置文件设置选项等。

  • Linux 一切皆文件

    Windows 是通过设备管理器来管理硬件，而 Linux 中所有内容都是以文件的形式保存和管理，包括硬件设备。Linux 的设备文件保存在 /dev/ 目录中。

  • Linux 不靠扩展名区分文件类型

    Windows 是依赖扩展名区分文件类型。比如，“.txt” 是文本文件、“.exe” 是执行文件、“.ini” 是配置文件等。而 Linux 则是靠权限位标识来确定文件类型，而且文件类型种类也不像 Windows 那么多。

    常见的文件类型只有普通文件、目录、链接文件、块设备文件等几种。

    Linux 的可执行文件就是普通文件被赋予了可执行权限。

    但是，在 Linux 中有些特殊文件还是要求写 “扩展名” 的。并不是说 Linux 需要扩展名来识别这些特殊文件，而是用来帮助管理员区分不同文件类型。

    1. 压缩包：Linux 中常见的压缩文件扩展名有 *.gz、*.bz2、*.zip、*.tar.gz、.tar.bz2 和 \.tgz 等。如果不写清楚扩展名，那么就不容易判断压缩包的格式。
    2. 二进制软件包：CentOS 中所使用的二进制安装包是 RPM 包，所有的 RPM 包都用 “.rpm” 扩展名结尾。
    3. 程序文件：Shell 脚本一般用 “*.sh” 扩展名结尾，还有用 ”.c“ 结尾的 C 语言文件等。
    4. 网页文件：网页文件一般使用 “*.html” 结尾，这是 web 服务器要求，和 Linux 无关。

  • Linux 中所有的存储设备都必须在挂载之后才能使用

    Linux 中所有的存储设备都有自己的设备文件名，这些设备文件必须在挂载之后才能使用，包括硬盘、U盘和光盘。

    挂载就是给这些存储设备分配盘符，在 Windows 中的盘符永英文字母表示，而在 Linux 中则是一个已经创建的空目录。将这些空目录称为挂载点（可以理解为 Windows 的盘符），把设备文件（如 /dev/sdb）和挂载点（已经创建的空目录）连接的过程就称为挂载。

  • Windows 下的程序不能直接在 Linux 中使用

    Linux 和 Windows 是不同的操作系统，可以安装和使用的软件也是不同的。
